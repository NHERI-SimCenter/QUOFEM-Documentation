UQ: Uncertainty QuantificationÂ¶

The first selection panel the user must select from and enter data into is the UQ tab. It is in this panel that the user selects the UQ Engine to use for performing the uncertainty quantification calculations. The UQ Engine provides algorithms for solving various types of uncertainty analysis and optimization problems.

The UQ Engine options currently available are Dakota, SimCenterUQ, and UCSD-UQ. Users can also configure quoFEM to use their own UQ methods and algorithms in the quoFEM workflow by selecting the CustomUQ option.

UQ features at-a-glance:

Forward Propagation
Forward propagation generates sample realizations of input random variables (RVs) and output quantity of interests (QoIs) to provide statistics such as mean, variance, skewness, and kurtosis.
* Forward Propagation in Dakota (Example1, Example2)
* Forward Propagation in SimCenterUQ (Example)
Global Sensitivity Analysis
Global sensitivity analysis is used to quantify contribution of each input RV to the uncertainty in an output QoI. Dakota engine provides classical non-parametric estimation based on smart sampling approach and SimCenterUQ engine provides probabilistic model-based approximation. See Dakota user manual and here for theory details.
* Global Sensitivity Analysis in Dakota (Example)
* Global Sensitivity Analysis in SimCenterUQ (Example1, Example2)
Reliability Analysis
Reliability Analysis is performed to estimate the probability of failure, i.e. the probability that a system response (QoI) exceeds a certain threshold level.
* Reliability Analysis in Dakota (Example)
Inverse Problem (Bayesian Calibration)
Inverse Problem (Bayesian Calibration) is used to calibrate model parameters probabilistically based on Bayesian inference. The probability distributions of the input parameters (RVs) are updated by experimental data. Theory details can be found in Dakota user manual and here
* Inverse Problem in Dakota
* Transitional Markov chain Monte Carlo (TMCMC) algorithm in UCSD-UQ (Example1, Example2)
Parameter Estimation (Deterministic Calibration)
Parameter Estimation (Deterministic Calibration) estimate the best parameter values of a simulation model that best fit the experimental data, using deterministic optimization algorithms, e.g. Gauss-Newton least squares
* Parameter Estimation in Dakota (Example1, Example2)
Surrogate Modeling
quoFEM app can be used to train a surrogate model model that substitutes expensive computational simulation models or physical experiments. Theory details can be found in here.
* Surrogate Modeling in SimCenterUQ (Example1, Example2)
* Probabilistic Learning on Manifolds (PLoM) in SimCenterUQ
Custom UQ
Custom UQ helps user to plug-in an user-defined UQ algorithm in SimCenter workflow.
* Configuring CustomUQ Engine in CustomUQ engine (Example)

Dakota UQ EngineÂ¶

This UQ engine utilizes the Dakota Software, a state-of-the-art research application that is robust and provides many methods for optimization and UQ, a selection of which we utilize in this application. Dakota provides the user with a large number of methods for different kinds of analyses. For this reason we have divided the methods into categories though a pull-down menu, as shown in figDakota. Once the category has been selected, a number of different methods are made available to the user.

The following categories are available:

  • 2.1.1.2. Global Sensitivity Analysis
  • 2.1.1.3. Reliability Analysis
  • 2.1.1.4. Parameter Estimation
  • 2.1.1.5. Inverse Problem

SimCenter UQ EngineÂ¶

The SimCenterUQ engine is a UQ engine developed in-house at the SimCenter that accommodates different UQ methods, which are organized into categories that can be accessed through a pull-down menu, as shown in figSimCenterUQ.

Currently the following category options are available:

  • 2.1.2.1. Forward Propagation
  • 2.1.2.2. Global Sensitivity Analysis
  • 2.1.3.1. Surrogate Modeling
  • 2.1.2.3. Probabilistic Learning on Manifolds (PLoM)

UCSD UQ EngineÂ¶

The UCSD-UQ engine is a module developed at the SimCenter in collaboration with UCSD. It provides algorithms for Bayesian estimation, which can be accessed through a pull-down menu, as shown in figure figUCSDUQ.

This module currently offers support for Bayesian estimation using the Transitional Markov chain Monte Carlo (TMCMC) algorithm:

  • 2.1.4.1. Transitional Markov chain Monte Carlo (TMCMC) algorithm

Custom UQ EngineÂ¶

The CustomUQ option enables users to switch out the UQ engine in the quoFEM workflow such that different methods and tools can be applied within the SimCenter framework with minimal effort on the part of the user. The CustomUQ option can be accessed as shown in figure figCustomUQ.

In order to use the CustomUQ engine option, two steps are required:

  • Configuring UQ tab to accept the required inputs

  • Adding UQ engine to customized UQ backend

These steps are described in more detail here:
